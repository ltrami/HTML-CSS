Capíitulo 2: Inserindo listas ordenadas, listas não-ordenadas e listas de definição

Aqui temos um exemplo de uma lista ordenada usando números, que é o padrão.

  1. Item 1
  2. Item 2
  3. Item 3

Para inserir listas ordenadas, utilizamos a tag <ol>, que significa Ordered List.

Podemos também usar letras maiúsculas e minúsculas para ordenar nossa lista:

  1. Item 1
  2. Item 2
  3. Item 3
  1. Item 1
  2. Item 2
  3. Item 3

Podemos usar também algarismos romanos, também usando caracteres maiusculos ou minúsculos:

  1. Item 1
  2. Item 2
  3. Item 3
  1. Item 1
  2. Item 2
  3. Item 3

É possível inserir listas não-ordenadas utilizando a tag <ul>. Lembrando que para inserir a tag em angled brackets num texto como este, é necessário usar ampersand lt; (indicando "less than", criando o símbolo de "menor que") e ampersand gt; (indicando "greater than", criando o símbolo de "maior que") em volta da tag. Do contrário, o editor reconheceria como uma tag e não como texto.

Um exemplo de lista não-ordenada:

  • Item 1
  • Item 2
  • Item 3

Pode-se alterar o símbolo usado para marcar os itens de uma lista não-ordenada. Para isso, utiliza-se o atributo type="" como atributo da tag <ul> da mesma forma que se usa em listas ordenadas. No entanto, ao invés de números e letras, usamos valores diferentes para o atributo type="". Estes valores são: disc, circle e square.

Exemplo de lista não-ordenada usando o valor disc para o atributo type:

  • Item 1
  • Item 2
  • Item 3

Exemplo de lista não-ordenada usando o valor circle:

  • Item 1
  • Item 2
  • Item 3

E por último, usando o valor square:

  • Item 1
  • Item 2
  • Item 3

Por fim, temos a lista de definição, que é inserida usando a tag <dl> (definition list) para abrir a a lista. Diferente das listas ordenadas e não-ordenadas, em que se usa <li> para incluir cada item, nas listas de definição usa-se a tag <dt> para indicar o termo (definition term) e <dd> para indicar a descrição do termo (definiton description).
